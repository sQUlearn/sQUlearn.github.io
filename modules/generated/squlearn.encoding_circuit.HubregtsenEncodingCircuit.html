class squlearn.encoding_circuit.HubregtsenEncodingCircuit(num_qubits: int, num_features: int, num_layers: int = 1, closed: bool = True, final_encoding=False)ÔÉÅ

Creates the data reuploading encoding circuit as presented in reference [1].

Example for 4 qubits, a 2 dimensional feature vector, 2 layers:

(Source code, png, hires.png, pdf)

The encoding can be optionally repeated at the end to make the previous rotations not redundant in a fidelity kernel setting. The circuit is closed by default, i.e. the last qubit is entangled with the first one.

Parameters:

  • num_qubits (int) ‚Äì Number of qubits of the encoding circuit

  • num_features (int) ‚Äì Dimension of the feature vector

  • num_layers (int) ‚Äì Number of layers (default:1)

  • closed (bool) ‚Äì If true, the last and the first qubit are entangled; not necessarily hardware efficient! (default: true)

  • final_encoding (bool) ‚Äì If True, the encoding is repeated at the end (default: False)

References

[1]: T. Hubregtsen et al., ‚ÄúTraining Quantum Embedding Kernels on Near-Term Quantum Computers‚Äù, arXiv:2105.02276v1 (2021).
